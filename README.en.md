# ColumbulaSort 🕊️

A high-performance C++ radix sort header-only library. Single header, drop-in use, 2–20× faster than `std::sort` in most scenarios.

All the code is generated by DeepSeek-V4 and hasn't been widely tested, so use it with caution.

---

## Quick Start

```cpp
#include "ColumbulaSort.h"
#include <vector>

int main() {
    ColumbulaSort cs;

    // Integers
    std::vector<int> arr = {5, 3, 8, 1, 9, 2, 7, 4, 6, 0};
    cs.sort(arr);                     // sort(vector&)
    cs(arr);                          // operator()(vector&)

    // Floating point
    std::vector<double> floats = {3.14, -2.71, 1.41, -0.58};
    cs.sort(floats);

    // Strings
    std::vector<std::string> strs = {"pear", "apple", "banana", "orange"};
    cs.sort(strs);

    // Sort by key
    struct Point { int x, y; };
    std::vector<Point> pts = {{5,1}, {3,4}, {8,2}};
    cs.sort(pts, [](const Point& p) { return p.x; });

    return 0;
}
```

### Supported types

| Type | Notes |
|------|-------|
| `int`, `unsigned int` | 32-bit, AVX2 accelerated |
| `short`, `unsigned short` | 16-bit, best speedup |
| `long long`, `unsigned long long` | 64-bit |
| `char`, `signed char`, `unsigned char` | Sorted by unsigned byte value |
| `float`, `double` | IEEE 754 bit-level total order |
| `std::string` | Lexicographic, faster for short/medium strings |
| Custom type + key func | All arithmetic key types supported |

### Build requirements

```bash
g++ -O3 -march=native -std=c++17 your_program.cpp -o your_program
```

Requires C++17. `-march=native` enables AVX2 for maximum performance.

## How it works

LSD (Least Significant Digit First) radix sort. Each pass processes 8 bits using stable counting sort.

### Key optimizations

- **Input-adaptive** — detects sorted/reverse/constant data and short-circuits instantly
- **SIMD** — AVX2-accelerated min/max analysis and index precomputation for 32-bit unsigned types
- **Tiered counting sort** — stack allocation for small N, alloca for medium N, multi-path dispatch for large N
- **Floating-point total order** — maps IEEE 754 bit patterns to a monotonic unsigned representation, handles sign bits and NaN
- **String acceleration** — packs short/medium strings into uint64_t multi-key sort
- **Thread-safe buffers** — `thread_local` vectors to avoid repeated allocation
